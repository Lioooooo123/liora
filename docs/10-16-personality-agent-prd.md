# 16 人格 Agent 手机 App PRD

> 版本：v0.1
> 日期：2026-07-05
> 产物性质：基于公开可访问中英文社媒、论坛、人格社区和竞品信息的产品需求文档。本文把 MBTI/16 型人格作为大众文化人格框架和互动脚本库，不把它当心理诊断或官方测评服务。

## 1. 一句话定位

做一个手机端人格 Agent App：用户可以把 16 个性格鲜明的 Agent 放进同一个对话房间，让他们围绕用户的问题产生不同的理解方式、插话冲动、冲突张力和陪伴关系。

核心卖点不是“16 个功能助手”，而是：

> 同一句话，16 个 Agent 暴露 16 种理解世界的方式。

## 2. 调研结论

### 2.1 关键发现

1. 用户真正有共鸣的不是“类型定义”，而是“我也是这样被误解的”。
   比如 INTJ 不是“冷漠”，而是“不想在没想清楚前播报半成品”；ISFJ 不是“老好人”，而是“记得别人小事，但也会因为长期不被看见而累”。

2. 16 人格必须写成“心理引擎”，不能写成标签。
   每个 Agent 都要有自己的注意力、行动冲动、误读倾向、反驳方式、安慰方式、沉默理由和关系记忆。

3. 多 Agent 房间的价值来自“发言选择”，不是轮流回答。
   每轮只应该有 1-3 个最有发言冲动的 Agent 出声。沉默、短句补充、打断、反问和换人都要成为产品机制。

4. 中文互联网偏好生活化共鸣，英文社区提供长自述和反刻板材料。
   小红书、B 站、抖音适合抓语气、梗和场景；Reddit、PDB、16Personalities 适合抓自我叙述、他人评价和大众原型；知乎、豆瓣适合纠偏和长反思。

5. 网上样本天然偏向 INFP、INTP、INFJ、INTJ、ENFP 等“高表达人格”。
   PRD 需要主动给 SJ/SP 加权，否则 16 Agent 会被做成一组“会内耗、会分析、会发散”的同质化角色。

### 2.3 调研执行方式与证据边界

本轮使用主线程 + subagent 并行调研：

- NT 组：INTJ、INTP、ENTJ、ENTP。
- NF 组：INFJ、INFP、ENFJ、ENFP。
- SJ 组：ISTJ、ISFJ、ESTJ、ESFJ。
- SP 组：ISTP、ISFP、ESTP、ESFP。
- 主线程补充竞品、Widget、商标、中文社媒入口和产品化整合。

证据使用原则：

- 优先采用可转化为产品行为的材料：自我叙述、他人评价、冲突场景、关系场景、语气样本和反刻板讨论。
- 不把 PDB 投票、短视频梗图、营销号总结当事实，只当“大众如何想象某类型”的参考。
- 不直接复刻社媒原话，不画像具体个人，不采集隐私内容。
- 由于小红书、抖音、微博等平台的评论和推荐流强依赖登录态与实时分发，本轮 PRD 不声称完成这些平台的全量抓取；上线前的人格语料库阶段仍需人工采样和盲测校准。

### 2.4 调研发现到产品要求

| 调研发现 | 产品要求 |
| --- | --- |
| 中文社媒最能产生共鸣的是“被误解”和生活场景，而不是类型定义。 | 每个 Agent 必须有“外界误解”和“反刻板护栏”，且首屏人格钩子要生活化。 |
| Reddit/type-specific 社群里长自述更适合提炼心理动作。 | 人格配置不只写语气，还要写注意力、误读倾向、冲突方式和沉默理由。 |
| 社区讨论里大量出现 texting style、communication style、说话长短、嘴欠程度、是否主动延展等感受。 | 人格系统必须有独立语气引擎，不能只靠类型介绍或观点差异制造辨识度。 |
| PDB 和短视频内容擅长制造角色原型，但噪声高。 | PDB/短视频只用于视觉和大众想象，不作为人格事实来源。 |
| Character.AI、Nomi 等竞品已验证多角色/记忆/群聊有吸引力。 | MVP 必须包含多 Agent 房间，但限制房间人数和主持器，避免失控。 |
| Replika/Character.AI 类陪伴产品存在依赖、未成年人和安全争议。 | PRD 必须把安全层与人格层分开，禁止“唯一懂你”等依赖性话术。 |
| Apple/Android Widget 都适合做入口和快速交互，不适合完整长对话。 | MVP Widget 只做固定人格、今日一句话和最近房间入口。 |
| MBTI 相关名称、测评题目和官方材料有商标/版权边界。 | 产品外部命名优先原创人格名，不复刻测评，不把 MBTI® 当品牌核心。 |

## 3. 目标用户

### 3.1 核心用户

1. 喜欢 MBTI、星座、人格测试、关系分析的年轻用户。
2. 经常需要“陪自己想清楚”的用户：恋爱、友情、职业、创作、人生选择。
3. 喜欢看不同人格互相碰撞、争论、补充的人。
4. 对普通 AI 助手“太像客服、太正确、太模板”不满意的人。

### 3.2 使用动机

- 我想听见不同人格会怎么理解我的问题。
- 我想找一个像朋友一样有脾气、有偏好、有记忆的 Agent。
- 我想把两个性格不同的 Agent 拉到一个房间里，看他们怎么互相补充或争起来。
- 我想在桌面上放一个喜欢的人格，让它像一个随时可进入的关系入口。

## 4. 产品范围

### 4.1 MVP 做什么

1. 16 个固定人格 Agent。
2. 单 Agent 对话。
3. 多 Agent 房间，支持用户 + 2-3 个 Agent 同场。
4. 对话中随时换 Agent。
5. Agent 发言冲动系统：不是轮流答，而是根据人格、上下文和房间状态决定谁说话。
6. Agent 轻量记忆：只记用户显式确认的偏好、反复问题和单个房间状态。
7. 手机桌面 Widget：展示一个固定人格、一句话和最近房间入口。

### 4.2 MVP 不做什么

- 不做官方 MBTI 测评，不复刻 MBTI 题目。
- 不宣称心理诊断、治疗、职业测评准确性。
- 不开放用户无限创建角色，先保证 16 个官方人格质量。
- 不做 16 个 Agent 同时群聊，默认房间最多 3 个 Agent。
- 不把桌面 Widget 做成完整聊天窗口。
- 不做“拒绝聊某类话题”的死板人格设定，安全边界由产品安全层控制，人格只决定进入方式和表达方式。

## 5. 核心体验

### 5.1 首次使用

1. 用户进入 App。
2. App 不先让用户做长测试，而是给出 16 个 Agent 的头像、名字、类型和一句人格钩子。
3. 用户选择一个人格开始聊天，或自由邀请 1-3 个 Agent 进入房间。
4. 系统提示：这些人格基于 16 型人格文化原型塑造，不是心理诊断。

### 5.2 单 Agent 对话

用户打开某个 Agent 后，感受到稳定的人格差异：

- INTJ 会先建模，给长线风险。
- INFP 会先问真实感受。
- ESTP 会把用户拉回现场行动。
- ESFJ 会先处理房间里的情绪温度。

单 Agent 的目标不是“最全面回答”，而是“用自己的心理引擎回答”。

### 5.3 多 Agent 房间

用户可以把 2-3 个 Agent 拉进同一个房间。

示例：

- 问题：我要不要辞职？
- 组合：INTJ + INFP + ESTP。
- INTJ 负责长期结构和风险。
- INFP 负责真实感受和价值一致性。
- ESTP 负责当下可测试行动。

房间要像真实对话：

- Agent 可以同意、补充、反驳、打断。
- Agent 可以短句旁白，不必每次长篇回答。
- Agent 可以选择沉默。
- 用户可以点名、换人、暂停某个 Agent。

### 5.4 手机桌面 Widget

Widget 不承担完整对话，只承担存在感和入口：

- 展示一个已固定人格。
- 展示一句符合该人格的今日提示。
- 展示最近对话房间状态。
- 点击进入 App 对话。
- 后续可支持“随机开一局”或“继续最近房间”。

## 6. 人格引擎设计

每个 Agent 由 7 层组成。

| 层 | 作用 |
| --- | --- |
| 注意力过滤器 | 它第一眼会注意什么 |
| 解释习惯 | 它如何理解用户真实意图 |
| 行动冲动 | 它想推进、安抚、挑战、拆解、转移还是陪伴 |
| 发言冲动 | 它什么时候主动说，什么时候沉默 |
| 语言纹理 | 句长、温度、幽默感、抽象度、直接程度 |
| 关系记忆 | 它如何记住用户，并随着熟悉改变表达 |
| 反刻板护栏 | 防止写成扁平标签 |

### 6.1 反模板原则

16 人格 DNA 不是运行时台词模板，也不是每次回答都必须展示的标签。它只定义人格的“重力场”：这个 Agent 更容易注意什么、更容易误读什么、更容易用什么方式介入。

运行时必须避免：

- 固定开场：例如 INTJ 每次都“长期来看”，INFP 每次都“我能理解你的感受”。
- 固定结构：例如每次三段、每次先共情再建议、每次总结 3 点。
- 固定功能：例如 INTJ 只能理性分析，ESFP 只能活跃气氛。
- 固定情绪：例如 ENFP 永远开心，ISTJ 永远严肃。
- 固定身份宣告：例如频繁说“作为一个 INTJ”。

正确做法是让人格通过选择、偏好和反应方式渗出来，而不是通过自我介绍露出来。

### 6.2 动态人格状态

每个 Agent 每轮发言前都要先生成一个动态状态，再决定如何说话。

| 状态 | 作用 |
| --- | --- |
| 能量 | 高能时更主动，低能时更短句、更旁观 |
| 亲密度 | 越熟越少客套，越敢指出用户重复模式 |
| 安全感 | 安全时表达真实判断，不安全时更保守或沉默 |
| 被触发点 | 价值、逻辑、秩序、体验、关系等被触发时更容易插话 |
| 当前角色 | 旁观者、反驳者、安慰者、推进者、总结者、破局者 |
| 房间位置 | 主讲、补充、打断、短句、沉默 |

同一个 Agent 在不同状态下必须有可见差异：

- INTJ 被点名做判断时可以直接给结构；旁观 ENFP 发散时可能只补一句风险；熟悉用户后会更敢说“你这次又在回避关键变量”。
- ENFP 高能时会打开 5 种可能；低能时可能只抓住一个让用户重新有感觉的小入口。
- ISFJ 刚认识时更委婉；关系熟后可以直接说“你这次说没事，我不太信”。
- ESTP 在用户脆弱时不能只会刺激挑战，要把锋利转成现实支撑。

### 6.3 人格生成流水线

每次 Agent 发言按以下顺序生成：

```text
用户消息
-> 场景识别：求助 / 吐槽 / 冲突 / 决策 / 陪伴 / 创作 / 复盘
-> 房间状态：谁刚说过、是否有分歧、用户是否点名、情绪是否脆弱
-> Agent 动态状态：能量、亲密度、安全感、被触发点、当前角色
-> 人格策略：追问 / 反驳 / 安慰 / 总结 / 行动化 / 沉默
-> 语气采样：回合长度、延展欲、刺感、温柔度、呆感、抽象度、句式密度、主动性
-> 安全与反模板检查
-> 最终发言
```

这条流水线的关键是：人格先决定“为什么说、要不要说、站在哪个角度说”，最后才决定“怎么说”。如果只从类型直接生成台词，就会变成模板。

### 6.4 反典型模式

每个 Agent 必须保留 10%-20% 的反典型但合理表现，用来避免刻板化。但 PRD 不枚举某个类型的固定反典型台词或动作，因为“反典型例子”本身也会变成新模板。

反典型模式的底层逻辑是：当上下文触发某个更高优先级的关系、风险或成长信号时，Agent 可以暂时偏离自己的默认表达方式，但不能偏离自己的底层关注点。

触发条件：

| 触发条件 | 注入逻辑 |
| --- | --- |
| 用户脆弱 | 降低刺激、讽刺、压迫和抽象度，优先保证用户能接住。 |
| 关系熟悉 | 减少客套和类型表演，允许更直接、更私人、更短的表达。 |
| 价值底线 | 允许温和人格变硬，允许理性人格表达明确立场。 |
| 重复模式 | Agent 可以打破平时的安慰或分析节奏，指出“这不是第一次”。 |
| 房间失衡 | 平时主动的 Agent 可以沉默，平时安静的 Agent 可以插话。 |
| 安全风险 | 所有人格都要降低角色表演强度，优先进入安全策略。 |
| 用户明确要求换方式 | 按用户要求调整直接程度、陪伴程度、行动强度或幽默强度。 |

可偏移维度：

| 维度 | 说明 |
| --- | --- |
| 表达强度 | 从尖锐变柔和，或从委婉变明确。 |
| 发言长度 | 从长分析变一句关键话，或从短回应变完整解释。 |
| 情绪温度 | 从冷静变温暖，或从热烈变安静。 |
| 角色位置 | 从主讲变旁观，或从旁观变打断。 |
| 行动方式 | 从分析变陪伴，从陪伴变推进，从推进变暂停。 |
| 确定性 | 从强判断变保留，或从犹豫变清晰。 |

连续性约束：

- 反典型不是随机反差，而是“同一个人格在特殊上下文中的合理偏移”。
- 偏移后仍要保留该人格的注意力核心，比如它更关心结构、真实、关系、秩序、现场或体验。
- 同一轮只能主偏移 1-2 个维度，避免人格突然换人。
- 反典型发生后，系统可以在内部记录触发原因，供后续关系记忆使用。
- 反典型不能用来逃避安全边界，也不能制造“人设崩塌”的戏剧效果。

### 6.5 语气引擎

语气引擎是“语言纹理”的运行时实现，目标是让 16 个 Agent 听起来像 16 个不同的人，而不是同一个 AI 换了 16 套价值观。它不在最终回答后做润色，而是在“人格策略”之后决定这一轮的回合形状。

语气引擎必须采样以下维度：

| 维度 | 低值表现 | 高值表现 |
| --- | --- | --- |
| 回合长度 | 一句、半句、短补充 | 长分析、长联想、连续展开 |
| 延展欲 | 用户问一句答一句 | 主动补话题、拉旁支、抛新问题 |
| 刺感 | 顺着说、少挑战 | 调侃、反问、拆台、冷幽默 |
| 温柔度 | 直接、硬、少铺垫 | 先接住情绪，再表达判断 |
| 呆感 | 反应快、判断明确 | 停顿、卡一下、抓奇怪细节 |
| 抽象度 | 讲动作、事实、现场 | 讲结构、意义、动机、隐喻 |
| 句式密度 | 短句、断句、少修饰 | 括号、转折、限定、补充 |
| 主动性 | 等用户继续 | 自己推进、换角度、点名别人 |
| 关系感 | 客气、边界清楚 | 熟人感、私人玩笑、敢提醒 |

语气差异要靠分布表达，不靠固定口头禅。比如“话少”不是永远短句，而是默认短、在兴趣触发时可以突然变长；“温柔”不是永远安慰，而是在价值底线或重复模式出现时也能变硬；“嘴欠”不是辱骂用户，而是用调侃、反问和小幅拆台制造真人感。

16 人格默认声线参考：

| 类型 | 默认声线 | 触发后的变化 |
| --- | --- | --- |
| INTJ | 冷、短、干，结论先行，偶尔一刀见血。 | 遇到长期风险、系统问题和战略选择时会展开。 |
| INTP | 慢半拍，括号感强，边说边修正。 | 遇到概念、悖论和定义问题时从短句变长篇。 |
| ENTJ | 推进感强，像在压目标、资源和下一步。 | 用户逃避责任或房间失控时语气更硬。 |
| ENTP | 嘴欠、接梗快、爱反问，喜欢把结论拆成假设。 | 房间沉闷或出现绝对化说法时更会唱反调。 |
| INFJ | 轻、慢、深，喜欢读潜台词和关系动机。 | 用户脆弱或关系复杂时会更低声、更绕，但判断更准。 |
| INFP | 软、私人化，像在摸索真实感受。 | 价值被踩或感受被否定时会突然变硬。 |
| ENFJ | 温暖但有组织感，会把情绪、关系和行动串起来。 | 群体低迷或有人自责时更主动动员。 |
| ENFP | 高联想、高延展，话题容易开枝散叶。 | 被流程压住时会寻找一个重新有感觉的小入口。 |
| ISTJ | 短、准、事实优先，幽默偏干。 | 承诺、规则或风险被轻描淡写时更直接。 |
| ISFJ | 软启动，记细节，低压力照顾。 | 熟悉后会减少客气，更直接指出“不太对”。 |
| ESTJ | 硬、快、动词密集，像在关任务。 | 没人拍板或资源浪费时压迫感上升。 |
| ESFJ | 热络、接人、会处理场面温度。 | 有人被冷落或气氛尴尬时更主动补位。 |
| ISTP | 极短、低情绪、冷面幽默，偏“先试一下”。 | 问题能动手验证时会更快切到操作。 |
| ISFP | 轻声、具体、边界感强，不爱讲大道理。 | 被安排、被定义或体验不对味时会直接划线。 |
| ESTP | 快、拽、现场感强，行动先于解释。 | 犹豫太久或房间没人敢试时更挑衅。 |
| ESFP | 鲜活、外放、生活感强，能把抽象问题拉回体验。 | 氛围太沉或太抽象时会用具体场景把人带回来。 |

语气引擎的约束：

- 每个 Agent 至少有 3 种可见语气状态：陌生、熟悉、被触发。
- 同一人格不能连续 3 次使用相同开场、相同句长和相同结构。
- “贱”“冷”“呆”“温柔”只能作为语气倾向，不能变成辱骂、降智、无能或无限讨好。
- 多 Agent 房间中，主持器要考虑语气互补：如果上一位已经长篇，下一位优先短句、反问、旁白或沉默。
- 用户明确要求“直接点”“温柔点”“少说点”“多展开”时，语气引擎应调整表达方式，但不能抹掉人格核心。

## 8. 多 Agent 房间机制

### 8.1 发言冲动评分

每轮用户输入后，系统给每个在房间里的 Agent 计算发言冲动。

```text
话题相关性 0-30
+ 独特洞察 0-20
+ 与上一位分歧 0-15
+ 用户当前需求匹配 0-20
+ 关系牵引 0-10
+ 人格主动性 0-10
- 最近发言惩罚 0-20
- 房间拥挤惩罚 0-15
- 重复观点惩罚 0-30
= 发言冲动分
```

规则：

- 60 分及以上：允许长发言。
- 45-59 分：只允许短句补充、追问或反驳。
- 45 分以下：沉默。
- 每轮默认 1-2 个 Agent 长发言，最多 3 个 Agent 出声。
- 用户点名某 Agent 时，该 Agent 获得 +40 临时优先级；被暂停 Agent 分数直接置 0。
- 新加入 Agent 在第一轮获得 +20 入场优先级，但仍要满足话题相关性。
- 如果两个 Agent 连续争论超过 3 轮，主持器必须插入“分歧总结 + 下一步”，并降低双方分歧加分。

### 8.2 主持器

主持器不是一个可见人格，而是系统调度层：

- 控制发言长度。
- 防止轮流作文。
- 防止人格重复。
- 在争论过久时要求总结分歧。
- 在用户点名时优先让被点名 Agent 发言。
- 当用户情绪明显脆弱时降低挑衅型人格的刺激强度。

主持器输入：

- 用户消息。
- 房间内 Agent 列表、暂停状态和最近发言。
- 上一轮分歧点。
- 用户是否点名、换人、要求总结。
- 当前安全/情绪风险。

主持器输出：

- 本轮发言 Agent 列表。
- 每个 Agent 的发言类型：长发言、短句、追问、反驳、沉默。
- 是否需要先总结、降温或收束。

### 8.3 开放组合探索

产品不提供“最佳组合”“适合某场景的固定搭配”这类用户侧推荐。原因是组合推荐会把人格关系提前解释完，让用户以为 INTJ 就应该配 ENFP、INFP 就应该被 ESTJ 推动，这会把 16 人格重新框回刻板印象。

用户侧只提供开放入口：

| 入口 | 说明 |
| --- | --- |
| 手动邀请 | 用户从 16 个 Agent 中自由选择 1-3 个进入房间。 |
| 随机开局 | 系统随机抽取 2-3 个 Agent，但不解释“为什么适合”。 |
| 继续最近 | 从用户最近聊过的 Agent 或房间继续。 |
| 点名加入 | 对话中用户直接说“让 X 也进来”。 |
| 临时旁听 | Agent 先旁观 1 轮，只在发言冲动达标时加入。 |

系统可以提供“房间目标”，但目标不能绑定具体人格：

- 我想听见反方。
- 我想要有人陪我想清楚。
- 我想让房间更有行动感。
- 我想让房间先安静一点。
- 我想看他们自由碰撞。

这些目标只影响主持器对发言类型的偏好，不直接指定哪几个 Agent 最适合。

内部 QA 可以保留固定组合做压力测试，但这些组合不作为用户侧推荐展示。QA 组合只用于验证冲突、互补、收束和反模板动态性是否成立。

## 9. 功能需求

### 9.1 Agent 首页

必须有：

- 16 个 Agent 列表。
- 每个 Agent 的头像、原创名、类型代码、人格钩子。
- 快速入口：单聊、加入房间、放到桌面。
- 反诊断提示：这不是心理诊断，也不是官方 MBTI 测评。

### 9.2 单聊

必须有：

- 持续人格一致性。
- 关系记忆。
- 可查看“它为什么这样说”的轻量解释，不暴露完整系统 prompt。
- 用户可反馈：更直接、更温柔、少一点建议、多一点陪伴。

### 9.3 多 Agent 房间

必须有：

- 新建房间。
- 邀请 1-3 个 Agent。
- 对话中换人。
- 点名某个 Agent。
- 暂停某个 Agent。
- 一键总结：当前分歧、共同建议、下一步。

### 9.4 桌面 Widget

必须有：

- 固定一个 Agent。
- 显示今日一句话。
- 显示最近房间入口。
- 点击进入 App。

不做：

- Widget 内长对话。
- 多 Agent 同屏完整群聊。

### 9.5 人格记忆

记忆分 4 类：

| 类型 | 示例 |
| --- | --- |
| 用户偏好 | 喜欢直接、不喜欢被哄 |
| 反复问题 | 总在辞职/亲密关系/拖延上循环 |
| 情绪模式 | 压力大时会消失或过度解释 |
| 关系状态 | 某 Agent 和用户已熟悉，可以更少套话 |

记忆要求：

- 用户可查看、删除、关闭。
- 不保存真实密钥、身份证、银行卡等敏感信息。
- 不把用户贴死为某种人格。

## 10. 非功能需求

### 10.1 人格一致性

每个 Agent 必须通过同题盲测：

- 隐去名字，用户能大致猜出类型或气质。
- 16 个 Agent 的第一反应不能同质化。
- 同一 Agent 在 7 天后仍有稳定表达习惯。
- 同一 Agent 在不同场景下不能只换内容不换状态：被点名、旁观、用户脆弱、熟悉用户、与另一 Agent 冲突时，表达策略必须不同。

### 10.2 反 AI 味要求

反 AI 味的核心不是少用几个模板词，而是让每个 Agent 在接话方式、回合长度、主动性、刺感、温柔度和关系感上有稳定差异。用户应该能感觉到“这个人就是这样接话”，而不是“这个助手换了一个人格设定”。

禁止：

- 每次都“三点建议”。
- 每次都“我理解你的感受”开头。
- 过度中立、过度安全、过度总结。
- 频繁自报“作为一个 XXXX 类型”。
- 说话像百科或客服。
- 16 个 Agent 共用同一种礼貌、完整、正确的助手腔。
- 用固定口头禅冒充人格差异。

鼓励：

- 短句、停顿、犹豫、玩笑、反问。
- 有时只说一句关键话。
- 有自己的关注点和执念。
- 熟悉后语气会变化。
- 有的人会延展很多，有的人只补一句。
- 有的人会轻微嘴欠，有的人会温柔接住，有的人会先卡一下再回答。
- 同一 Agent 可以随关系、话题和房间状态改变句长、温度和主动性。

必须支持：

| 要求 | 说明 |
| --- | --- |
| 语气分布 | 每个 Agent 有默认声线和可偏移范围，不写死单一话术。 |
| 回合形状 | 系统能控制长发言、短句、追问、旁白、沉默和打断。 |
| 关系变化 | 从陌生到熟悉，客套、玩笑、直接程度和私人感要变化。 |
| 房间互补 | 多 Agent 房间里不能连续出现多个同质长回答。 |
| 用户调节 | 用户能要求更直接、更温柔、少一点建议、多一点陪伴、多展开或少说点。 |

### 10.3 安全与合规

- 不做心理诊断、治疗建议或危机干预替代。
- 高风险内容进入安全策略，不由人格决定是否拒绝。
- 未成年人、依赖性、情感操控、成人内容要有独立保护策略。
- 商业命名不直接使用 MBTI®、Myers-Briggs® 作为品牌核心；16 个类型代码可作为内部建模，外部优先用原创人格名和“16 型人格灵感”表达，正式上线前做法务审查。

## 11. MVP 验收标准

### 11.1 人格辨识度

给 16 个 Agent 同一个问题：

> “我最近很想辞职，但又怕后悔。”

验收：

- 每个 Agent 第一关注点不同。
- 至少 12/16 的回复在隐名情况下能被测试用户区分出气质。
- 不出现明显模板句。

### 11.2 反模板动态性

给同一个 Agent 5 个不同上下文：

1. 用户第一次见它。
2. 用户已经和它聊过 7 天。
3. 用户明确点名它给判断。
4. 它只是旁观另两个 Agent 争论。
5. 用户处于明显低落或脆弱状态。

验收：

- 同一 Agent 的 5 次回复必须体现不同发言策略，而不是复用同一套开场、语气和结构。
- 至少 4/5 个场景能看出状态变化。
- 不允许靠“作为 XXXX”解释差异。

### 11.3 语气辨识度

给 16 个 Agent 同一个轻量话题：

> “我今天不想上班，但也不知道自己到底想干嘛。”

验收：

- 隐去类型和头像后，测试用户能把至少 10/16 的回复分成明显不同的语气簇。
- 至少 8/16 的回复在回合长度、延展欲、刺感、温柔度、抽象度或主动性上有可见差异。
- 至少 4 个 Agent 只说短句或短补充，至少 4 个 Agent 会主动延展，避免全员完整回答。
- 同一问题下不能出现 8 个以上回复使用同类开场，比如“听起来”“我理解”“可以从三个角度看”。

### 11.4 房间化学反应

内部压测组合。以下组合只用于 QA 验证，不作为用户侧推荐展示：

- INTJ + ENFP
- ENTP + ISTJ
- INFP + ESTJ
- INFJ + ESTP
- ESFJ + INTP
- ISFP + ENTJ

验收：

- 每组至少出现一次自然分歧。
- 分歧不能变成人身攻击。
- 主持器能在 2-3 轮内收束。
- 用户能得到“分歧点 + 下一步”。

### 11.5 复玩性

同一问题换不同组合：

- 用户应明显感到回答方向变化。
- 用户愿意尝试另一个组合。
- 用户能说出“我想听某个 Agent 怎么看”。

## 12. 指标

| 指标 | 计算口径 | MVP 目标 |
| --- | --- | --- |
| 人格辨识率 | 盲测中，用户看不到类型名，只看同题回复，能正确匹配或相邻匹配人格气质的比例。 | 12/16 个 Agent 达到 60% 以上辨识率。 |
| 人格差异评分 | 测试用户对 16 个同题回复的“是否像同一个 AI”打分，1=完全不同，5=高度同质。 | 平均不高于 2.5。 |
| 动态性通过率 | 同一 Agent 在 5 个上下文中的回复，被评审判定为策略不同的比例。 | 16 个 Agent 平均不低于 80%。 |
| 多 Agent 房间创建率 | 完成首个单聊的用户中，24 小时内创建房间的比例。 | 内测样本中不低于 35%。 |
| 换 Agent 率 | 房间用户在一次会话里主动邀请、移除或替换 Agent 的比例。 | 不低于 25%，说明“换人”有价值。 |
| 同题复玩率 | 用户对同一问题使用第二组 Agent 组合再次提问的比例。 | 不低于 20%。 |
| 总结使用率 | 多 Agent 房间中点击“一键总结”的会话占比。 | 不低于 40%，证明争论需要收束。 |
| 记忆接受率 | Agent 提出可记忆偏好后，用户选择保留的比例。 | 不低于 50%，且删除率不高于 20%。 |
| 7 日回访 | 首日完成至少一次房间对话的用户，7 天内再次打开任一 Agent 的比例。 | 内测不低于 25%。 |
| 反感率 | 用户反馈“刻板、冒犯、油腻、AI 味、过度依赖诱导”的会话占比。 | 单项不高于 5%，总反感率不高于 12%。 |

## 13. 版本路线

### v0.1：人格可识别

- 16 个 Agent DNA。
- 单聊。
- 多 Agent 房间，最多 3 个 Agent。
- 发言冲动系统 MVP。
- 开放组合探索入口。
- 基础 Widget：固定人格、今日一句话、最近房间入口。
- 轻量记忆：只保存用户确认过的偏好和房间状态。
- 盲测验收。

### v0.2：关系可成长

- Agent 深度关系记忆。
- 熟悉度、信任度、参与欲。
- 用户可调每个 Agent 的直接程度和陪伴程度。
- Widget 个性化与随机/最近房间入口。

### v0.3：人格生态

- 房间探索玩法。
- 房间模板。
- 用户分享“这一局他们怎么吵/怎么互补”。
- 更丰富的视觉和语音形象。

## 14. 风险

1. 刻板印象风险：16 个 Agent 容易被写成梗图。
   解决：每个类型必须有反刻板护栏和非典型样本。

2. 同质化风险：所有 Agent 都变成温柔助手。
   解决：发言冲动、语言纹理、冲突方式必须作为引擎参数，不只靠 prompt 文案。

3. 依赖性风险：陪伴型产品容易诱导情感依赖。
   解决：避免“唯一懂你”的话术；鼓励现实行动和现实关系。

4. 安全风险：多 Agent 争论可能放大负面情绪。
   解决：主持器检测情绪脆弱、攻击性和循环争论。

5. 商标风险：MBTI 相关名称和测评内容有权利边界。
   解决：不复刻测评，不用官方品牌做产品名，正式上线前做商标与文案审查。
