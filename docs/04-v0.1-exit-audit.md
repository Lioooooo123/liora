# Liora v0.1 Exit Audit

这份文档把当前长期目标收敛为有限、可验证的 v0.1 能力底座。目标结束只看这里的 P0 清单和验收脚本，不再把 Claude Code / Kimi Code 的所有高级能力无限并入 v0.1。

## 结束规则

当前 active goal 可以结束，当且仅当同时满足：

1. `docs/05-mvp-exit-benchmark.md` 的 v0.1 范围没有新增 P0 缺口。
2. 下方 P0 evidence matrix 每一项都有当前代码、测试、smoke 或 release artifact 证据。
3. 从干净且已推送的 `main` 分支运行 `./scripts/v0.1-exit-audit.sh "$PWD"` 通过。
4. 最新 release tarball 通过 `scripts/release-smoke.sh`，且 README 能说明安装和基础使用方式。
5. `implementation-notes.md` 记录了所有本轮为达成 v0.1 做出的取舍、风险和 v0.2 延后项。

开发中允许运行：

```sh
./scripts/v0.1-exit-audit.sh --skip-git-clean "$PWD"
```

但这只证明功能路径可用，不证明目标可以结束。

## v0.1 Non-Goals

这些不阻塞当前 active goal 结束，进入 v0.2+：

- Mac 原生 App、白板闪念、角色房间、Live2D 或完整二次元皮肤。
- Bubble Tea 全屏多面板 TUI；v0.1 只要求 line-based TUI 稳定解释状态和操作。
- Docker sandbox 默认化；v0.1 只要求 Docker executor 可配置，默认 patch mode 保护真实 workspace。
- 逐步授权弹窗和长期后台守护安全策略；v0.1 采用 task 级 approve/deny。
- 多 agent 调度、长期任务队列、自动记忆抽取、embedding 搜索、计费统计和复杂模型路由。
